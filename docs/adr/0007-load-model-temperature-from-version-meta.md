# ADR-0007: model temperature 로딩 소스를 활성 버전 메타로 고정

## Created At

2026-02-23 14:52 KST

## Status

Confirmed

## Context

파이프라인 실행 시 `model/temperature` 값을 어떤 소스에서 읽을지에 대한 기준이 불명확해, 환경별 레지스트리 값과 버전 메타 값이 충돌할 가능성이 있었다.
DEV-022에서는 활성 버전 단위로 재현 가능한 실행 설정을 보장해야 하며, 런타임 결정값이 버전 산출물과 분리되면 동일 버전 재실행 시 결과 일관성이 깨질 수 있다.
또한 운영 중 레지스트리 값 변경이 과거 버전 동작에 영향을 주면 추적성과 롤백 안정성이 저하된다.

## Decision

`model/temperature`는 레지스트리 값이 아니라 활성 버전의 `{version}_meta.yaml`에서 로딩하는 방식으로 결정한다.

## Rationale

활성 버전 메타를 단일 소스로 사용하면 버전 산출물만으로 실행 파라미터를 재구성할 수 있어 재현성과 감사 가능성이 높아진다.
대안으로 레지스트리 값을 우선 사용하는 방안을 검토했지만, 운영 중 동적 변경이 과거 버전 실행 의미를 바꿀 수 있어 버전 불변성 요구와 충돌해 기각했다.
또 다른 대안으로 레지스트리와 메타를 병합하는 방안을 검토했지만, 우선순위 규칙과 충돌 처리 복잡도가 증가해 운영 오류 가능성이 커지므로 기각했다.
이 결정의 트레이드오프로 중앙 제어의 즉시성은 줄어들지만, 버전 기준의 예측 가능성과 회귀 분석 용이성이 더 중요한 요구사항이므로 수용한다.
